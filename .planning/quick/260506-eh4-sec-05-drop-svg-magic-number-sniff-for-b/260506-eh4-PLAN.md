# Plan 260506-eh4: harden blog upload validation

## Goal

Resolve **SEC-05** (CONCERNS.md #18):
- Remove `image/svg+xml` from the upload allow-list (SVGs can XSS via inline `<script>` when served same-origin).
- Sniff the file's first ~12 bytes against magic numbers for JPEG / PNG / GIF / WebP. Reject anything that doesn't match.
- Derive the saved file's extension from the sniffed type, not from `path.extname(file.name)`.

## Tasks

Single atomic commit touching two files.

**`src/lib/upload.ts`**
- Implement `sniffImageType(buffer): "jpeg" | "png" | "gif" | "webp" | null` using inline byte checks (no new dep)
- Magic numbers:
  - JPEG: `FF D8 FF`
  - PNG: `89 50 4E 47 0D 0A 1A 0A`
  - GIF: `47 49 46 38 (37|39) 61`
  - WebP: `52 49 46 46 ?? ?? ?? ?? 57 45 42 50`
- Export `UnsupportedImageTypeError` for the route handler to catch
- In `saveUploadedFile`: sniff after reading bytes; throw `UnsupportedImageTypeError` if `null`; use `EXT_FOR[sniffed]` instead of `path.extname(file.name)`

**`src/app/api/blog/upload/route.ts`**
- Drop `"image/svg+xml"` from `allowedTypes`
- Wrap `saveUploadedFile` call in try/catch; map `UnsupportedImageTypeError` → 400, anything else → 500 with `console.error` log
