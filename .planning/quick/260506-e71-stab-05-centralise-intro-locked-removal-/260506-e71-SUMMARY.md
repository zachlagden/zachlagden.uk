# Quick Task 260506-e71: Centralise intro-locked lifecycle to home route only

Single atomic refactor moving `intro-locked` body class and `#initial-loader` element ownership from the root layout to the home route exclusively.

## What Was Done

Root layout (`src/app/layout.tsx`) previously applied `intro-locked` to the `<body>` className and rendered `<div id="initial-loader">` inside `<SessionProvider>` on every route. Non-home routes (blog, admin, auth/signin) had to mount `<ClearIntro>` or hand-roll a `useEffect` to undo this on every visit.

After this change:

- Root layout body className contains no `intro-locked` token and no `#initial-loader` div.
- `src/app/page.tsx` (home route, RSC) renders `<div id="initial-loader">` directly in SSR output — preserving pre-hydration loader coverage — and mounts `<HomeIntroBootstrap />`.
- `HomeIntroBootstrap` uses `useLayoutEffect` to add `intro-locked` before first paint and remove it on unmount (client-side nav away from home).
- `ClearIntro.tsx` deleted. `blog/layout.tsx` and `admin/layout.tsx` no longer import it. `auth/signin/page.tsx` no longer needs its manual `useEffect` cleanup.
- Adding any future top-level route requires zero intro-related code.
