<objective>
Install `knip` as a devDependency and capture a verbatim baseline of unused-code findings into `.planning/runbooks/KNIP-BASELINE.md` so v3 regression comparison has a known reference point. The baseline is captured BEFORE the Dependabot batches in Plan 05 land — that way the baseline reflects the actual v2-open codebase, not a post-bump moving target.

Purpose: Phase-5 success criterion 5 requires knip installed + baseline captured. Per D-08 the baseline is committed verbatim (no curation) — signal preservation > aesthetics. Future-Zach will diff raw output, not interpret summaries.

Output: One commit modifying `package.json` + `pnpm-lock.yaml` (knip in devDependencies, lockfile updated) and one new file `.planning/runbooks/KNIP-BASELINE.md` with the verbatim markdown report plus a capture-metadata header.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `pnpm dlx knip` → repo source | knip statically analyses TS/JS source; it does NOT execute project code. Read-only filesystem access. Low trust risk. |
| Captured baseline → committed git history | Verbatim output may include filename listings; no secrets, no env values are surfaced by knip's default reporter. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-06 | Information Disclosure | KNIP-BASELINE.md committed to public repo | accept | knip's markdown reporter lists file paths and unused export names — none of which are secrets. The repo is already public. No mitigation required. |
| T-05-07 | Tampering | knip 6.x has supply-chain risk like any npm dep | mitigate | Pinned via `pnpm add -D knip` (caret range) and registered in `pnpm-lock.yaml` with integrity hash. Bumps subject to the same Dependabot rules as any dev-dep (handled by Plan 05 Batch A). |
| T-05-08 | Denial of Service | knip incorrectly flags an actively-used file → developer deletes it → runtime regression | accept | The baseline is *captured*, not acted upon. Any actual cleanup based on knip findings is deferred to v3 (per Phase 5 scope). The baseline is read-only data. |
</threat_model>

    4. `pnpm install` exits 0 and produces no further pnpm-lock.yaml changes (`git diff pnpm-lock.yaml` is empty after a second `pnpm install`).
    5. `pnpm exec knip --version` exits 0 and prints the installed knip version.
    6. No other package.json key is modified: `git diff package.json | grep '^[-+]' | grep -v '^[+-]\s*"knip":' | grep -v '^[+-]\{3\}'` returns empty (only knip line added/removed).
  </acceptance_criteria>
  <verify>

       {paste the full contents of /tmp/knip-baseline-raw.md verbatim, no edits}
