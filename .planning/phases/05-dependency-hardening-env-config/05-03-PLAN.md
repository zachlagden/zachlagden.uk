<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Local Claude session ↔ Coolify API | Bearer token in `~/.claude/credentials/personal-infra.md` (chmod 600). Token grants write access to env vars on every Coolify-hosted app — full credential leakage = full compromise of every personal site. |
| GitHub user ↔ Claude session (PAT delivery) | The new fine-grained PAT is created in the GitHub UI and pasted into the Claude session ONCE. From there it must reach Coolify env without ever being written to disk in plaintext outside the Coolify API call. |
| Coolify env vars → Docker container | Standard Coolify behaviour; env vars injected at container start. Trust boundary is Coolify's own — out of scope for this plan beyond using the documented API. |
| Tracked git history ↔ public repo | The repo is public on GitHub. ANY secret committed is permanently leaked. Treat every Write/Edit on a tracked file as a public broadcast. |

      fi
