<objective>
Harden `.github/dependabot.yml` so future automated PRs cannot regress the two known break vectors in this codebase: the Framer Motion intro state machine (Pitfall 1) and the next-auth beta-to-beta OAuth contract (Pitfall 2). Belt-and-suspenders defence for the manual Dependabot batches in Plan 05 — even if a developer clicks "merge" on autopilot, the dangerous bumps are never proposed in the first place.

Purpose: Lock in the regression-prevention rails before the dependency-batch work in Plan 05 lands. This plan must merge first so any in-flight Dependabot PRs targeting `framer-motion` or `next-auth` are auto-cancelled by the new ignore rules.

Output: A single commit modifying `.github/dependabot.yml` with new `ignore:` entries for `next-auth`, `framer-motion`, and `pnpm`, plus group-rule cleanup so the ignored deps cannot leak back in via the existing `next-ecosystem` / `dev-dependencies` / `security-updates` groups.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub → repo (Dependabot) | Untrusted bot proposes changes to `package.json` + `pnpm-lock.yaml`; auto-merge is OFF in this repo, so a human reviews. The risk is human-on-autopilot merging a regression-class bump that passes lint+tsc but breaks runtime. |
| `.github/dependabot.yml` (config) → GitHub (control plane) | The YAML is the single source of truth for which deps Dependabot proposes; misconfiguration silently widens the merge surface. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | `.github/dependabot.yml` ignore rules | mitigate | Add `dependency-name: "next-auth"` with no `update-types` filter (= ignore ALL update types per Dependabot docs); add `dependency-name: "framer-motion"` with `update-types: ["version-update:semver-major", "version-update:semver-minor"]` (= patch-only proposals); confirm via `grep -A2 "next-auth\|framer-motion" .github/dependabot.yml` after edit. |
| T-05-02 | Elevation of Privilege | `next-auth` beta-to-beta bump silently lands → `requireAdmin()` returns 401 for legitimate admin (Pitfall 2) | mitigate | Full Dependabot ignore on `next-auth`; manual CVE-only bump per D-04, gated by AUTH-SMOKE-TEST.md (Plan 04). |
| T-05-03 | Denial of Service (UX) | `framer-motion` minor bump lands silently → intro state machine breaks → `intro-locked` body class persists → site appears frozen (Pitfall 1, regresses STAB-03/04) | mitigate | Patch-only Dependabot proposals; package.json range re-pinned to `~12.23.26` in Plan 05 Batch C; smoke test in three reduced-motion states per D-01 Batch C. |
| T-05-04 | Tampering | `pnpm` major bump (e.g. 10 → 11) silently lands → Coolify Corepack mismatch returns (commit 41657a0 was the fix) | mitigate | Add `dependency-name: "pnpm"` ignore for `version-update:semver-major`; document in PR description that pnpm bumps require re-validating Dockerfile Corepack invocation. |
| T-05-05 | Repudiation | Group rule `security-updates: patterns: ["*"]` could re-include ignored deps (Dependabot precedence ambiguity) | mitigate | Confirm Dependabot honours `ignore:` over `groups:` per docs; if the group rule still proposes, add `exclude-patterns: ["next-auth", "framer-motion"]` to the security-updates group. |

(Credential-leakage threats T-05-CRED-* live in Plan 03 — this plan only edits a YAML config with no secrets.)
</threat_model>
