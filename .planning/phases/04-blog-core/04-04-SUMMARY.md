## Decisions Made

**1. Use next-mdx-remote/rsc for MDX rendering**
- Rationale: Compatible with Next.js 15 App Router Server Components, renders MDX from database strings

**2. Unicode escape special characters in JSON-LD**
- Rationale: Prevents XSS attacks by escaping `<` and `>` as `\u003c` and `\u003e` in structured data

**3. IntersectionObserver for TOC active section**
- Rationale: Efficient way to track which heading is in viewport without scroll event listeners

**4. Hide TOC for posts with fewer than 3 headings**
- Rationale: TOC adds little value for short posts, saves screen space

**5. Extract headings via regex matching rehype-slug logic**
- Rationale: Ensures TOC links match IDs generated by rehype-slug plugin
