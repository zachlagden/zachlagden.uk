  <action>
Create the ThemeProvider wrapper component:

**src/components/providers/theme-provider.tsx:**
```typescript
"use client"

import * as React from "react"
import { ThemeProvider as NextThemesProvider } from "next-themes"

export function ThemeProvider({
  children,
  ...props
}: React.ComponentProps<typeof NextThemesProvider>) {
  return <NextThemesProvider {...props}>{children}</NextThemesProvider>
}
```

**Update src/app/layout.tsx:**

1. Import ThemeProvider from the new file
2. Add `suppressHydrationWarning` to the `<html>` tag - this prevents React hydration warnings when next-themes modifies the DOM before hydration
3. Wrap the body content with ThemeProvider using these props:
   - `attribute="class"` - applies dark mode via .dark class on html (Tailwind convention)
   - `defaultTheme="system"` - respects OS preference on first visit (THEME-03)
   - `enableSystem` - syncs with prefers-color-scheme changes
   - `disableTransitionOnChange={false}` - allows CSS transitions (user wants 150ms fade)

The layout structure should be:
```tsx
<html lang="en" dir="ltr" className={inter.variable} suppressHydrationWarning>
  <head>...</head>
  <body className={...}>
    <ThemeProvider
      attribute="class"
      defaultTheme="system"
      enableSystem
      disableTransitionOnChange={false}
    >
      <SmoothScrollProvider>{children}</SmoothScrollProvider>
    </ThemeProvider>
  </body>
  <GoogleAnalytics ... />
</html>
```

Keep the existing SmoothScrollProvider inside the ThemeProvider.

Update the body className to use theme-aware colors:
- Change `bg-neutral-50` to `bg-white dark:bg-[#0a0a0a]`
- This ensures the body background responds to theme changes
