<task type="auto">
  <name>Task 2: Create Auth.js configuration with GitHub provider and role extension</name>
  <files>
    - src/lib/auth.config.ts

    - src/types/next-auth.d.ts
    - src/app/api/auth/[...nextauth]/route.ts
  </files>
  <action>
    1. Create `src/types/next-auth.d.ts` - Type extensions:
       ```typescript
       import { DefaultSession } from "next-auth"

       declare module "next-auth" {
         interface Session {
           user: {
             id: string
             role: "admin" | "user"
           } & DefaultSession["user"]
         }
       }
       ```

    2. Create `src/lib/auth.config.ts` - Shared config for Edge compatibility:
       - Export `authConfig` with providers array containing GitHub
       - Include session configuration: strategy "database", maxAge 30 days
       - Include callbacks for session to add role based on `process.env.ADMIN_GITHUB_USERNAME`
       - This file is used by middleware (Edge runtime) so NO MongoDB imports

    3. Create `src/lib/auth.ts` - Full auth configuration:
       - Import and spread `authConfig`
       - Add MongoDBAdapter with clientPromise
       - Export `{ handlers, auth, signIn, signOut }` from NextAuth()
       - Use GitHub provider from "next-auth/providers/github"

    4. Create `src/app/api/auth/[...nextauth]/route.ts`:
       ```typescript
       import { handlers } from "@/lib/auth"
       export const { GET, POST } = handlers
       ```

    Admin detection logic in callbacks:
    - Compare `user.name` (GitHub username) with `process.env.ADMIN_GITHUB_USERNAME`
    - Default to "zachlagden" if env var not set
    - Set `session.user.role = "admin"` or `"user"` accordingly
    - Also add `session.user.id = user.id` for later use
  </action>
  <verify>
    - `pnpm build` succeeds (no TypeScript errors)
    - API route file exists at correct path
    - Types file properly extends Session interface
  </verify>
  <done>Auth.js configured with GitHub OAuth, MongoDB adapter, and admin role detection</done>
